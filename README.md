# Effective Mobile тестовое задание

Простое веб-приложение (Python backend) за реверс-прокси Nginx, развернутое в Docker-контейнерах.

## Как работает схема (nginx → backend)

Взаимодействие между компонентами происходит внутри изолированной Docker-сети. Бэкенд скрыт от внешнего мира и доступен только для Nginx.

1. **Пользователь (Хост)**: Отправляет HTTP-запрос на порт `80` вашего компьютера.
2. **Nginx контейнер**: Принимает запрос на порту `80`. Согласно конфигурации `proxy_pass`, он перенаправляет трафик на внутреннее имя сервиса `backend` внутри изолированной Docker-сети.
3. **Python контейнер**: Слушает внутренний порт `8080`. Обрабатывает запрос и возвращает текст.

**Схема взаимодействия:**
```text
[ Клиент ] --(Порт 80)--> [ Nginx (Reverse Proxy) ] --(Внутренняя сеть)--> [ Python бэкенд (8080) ]
```

## Использованные технологии**
- **Python 3.11-slim**: Официальный минималистичный образ.
- **Nginx (Stable-Alpine)**: Легковесный и стабильный веб-сервер.
- **Docker & Docker Compose**: Инструменты контейнеризации и оркестрации.

## Безопасность (Hardening)**
- **Non-root user**: Бэкенд запущен от имени пользователя appuser (не root).
- **Network Isolation**: Открыт только 80 порт Nginx. Порт бэкенда (8080) недоступен с хоста.
- **Healthcheck**: Настроена автоматическая проверка состояния бэкенда через Python.
- **Server Tokens**: Скрыта версия Nginx в заголовках ответа.



## Установка

### 1. Клонируйте репозиторий
```bash
git clone https://github.com/serejka-xaker/Web-Server-Nginx.git
cd Web-Server-Nginx
```

### 2. Создайте файл .env в корне проекта для настройки портов
```bash
cp .env.example .env
```

### 3. Заполните файл .env необходимыми значениями
```env
HOST_PORT=80
NGINX_VERSION=stable-alpine
```

### 4. Запустите проект через Docker Compose
```bash
docker-compose up --build -d
```

### Как проверить результат
```bash
curl http://localhost
```
